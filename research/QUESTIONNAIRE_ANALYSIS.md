## 2. Выявленные боли (ранжированы по важности)

### КРИТИЧЕСКИЕ (заказчик назвал в ТОП-3)

**1. Защита от out-of-stock (аут-сток)**
> «При резком приросте продаж быстро заканчивались остатки, а цена не повышалась, и поставка быстрая не представлялась возможной, из-за этого товар заканчивался раньше чем должен был.»

Суть: система должна автоматически **повышать цену при падении остатков**, чтобы замедлить продажи и дождаться поставки.

**2. Повышение акционных порогов (медианной цены)**
> «Всегда иметь высокие акционные пороги» (ответ на «магический» вопрос)

Суть: перед акциями нужно **заранее поднимать базовую цену**, чтобы при входе в акцию скидка была от более высокой медианы → меньше потеря маржи.

**3. Ценообразование на основе скорости продаж**
> «Скорость продаж 7 дней к предыдущим 14, динамика количества корзин и заказов за день к 7 дням»

Суть: система сравнивает **текущую скорость (7д) с предыдущей (14д)** и корректирует цену: если продажи падают — снижать, если растут — повышать.

### ВАЖНЫЕ

**4. WB меняет СПП и ломает витринную цену**
Нужен контроль финальной цены для покупателя с учётом СПП и WB-кошелька.

**5. Вход в акции в убыток**
Нужен прогноз маржинальности ДО входа в акцию + автоматический выход при нарушении маржи.

**6. Непонимание реальной маржинальности**
Нужен полноценный калькулятор юнит-экономики с учётом всех затрат.

**7. Товар залёживается — цена не снижается**
Обратная сторона out-of-stock: при падении спроса система должна снижать цену.

**8. Нехватка аналитики**
Текущий инструмент (Indeepa) не даёт достаточно аналитических данных.

## 3. Требования к MVP (первая версия)

Заказчик чётко назвал 4 функции для MVP:

### MVP-1: Прогноз маржинальности при входе в акцию
```
Показать: маржа при текущей цене → маржа при акционной цене
Входные данные: себестоимость, комиссия WB, логистика, возвраты, хранение, налоги, реклама
Результат: решение входить/не входить в акцию
```

### MVP-2: Защита от out-of-stock
```
Мониторинг остатков + скорости продаж
Если остатки < порога при текущей скорости → повышать цену
Замедлить продажи до момента поставки
```

### MVP-3: Поднятие медианной цены для акций
```
Перед акцией: заранее повышать базовую цену
Цель: увеличить акционный порог → меньше потеря маржи при входе в акцию
Стратегия работы с медианной ценой WB
```

### MVP-4: Ценообразование на основе динамики продаж
```
Формула: скорость_продаж_7д / скорость_продаж_14д = коэффициент тренда
Дополнительно: динамика корзин и заказов за день vs 7 дней
Если тренд ↑ → повышать цену
Если тренд ↓ → снижать цену
```

### MVP-5: Аналитика карточки (по образцу Indeepa)
Заказчик детально описал 4 графика + 3 борда:

**График 1: Динамика цен**
- Текущий % СПП
- Цена с СПП
- Макс. промо-цена

**График 2: Динамика основных параметров**
- Процент выкупа
- Скорость продаж (14 дней)
- Текущий уровень акционного бустинга
- Оборачиваемость (14 дней)
- Остаток общий

**График 3: Цена акции**
- Цена артикула
- Цена 1-й акции
- Цена 2-й акции
- Цена 3-й акции и т.д.

**График 4: Динамика цена-заказы**
- Цена со скидкой
- Цена с СПП
- Заказы (шт.)
- Отмены и возвраты

**Борд 1: Количество заказов по ценам до СПП**
— сколько заказов было по каждой цене (до СПП)

**Борд 2: Количество заказов по ценам с СПП**
— аналогично, но с учётом СПП

**Борд 3: Пиковые часы заказов**
— распределение заказов по часам суток

**Борд 4: Пиковые дни**
— распределение заказов по дням недели

## 4. Стратегии ценообразования (требуемые)

| # | Стратегия | Приоритет | Описание |
|---|-----------|-----------|----------|
| 1 | **Динамика продаж** | КРИТИЧЕСКИЙ | Скорость 7д/14д, корзины/заказы день/7д |
| 2 | **Защита от out-of-stock** | КРИТИЧЕСКИЙ | Повышение цены при низких остатках |
| 3 | **Акционный бустинг** | КРИТИЧЕСКИЙ | Поднятие медианной цены перед акциями |
| 4 | **Следование за конкурентом** | ВЫСОКИЙ | Быть дешевле/на уровне/дороже на X ₽/% |
| 5 | **Целевая маржа** | ВЫСОКИЙ | Минимальная маржинальность (не опускаться ниже) |
| 6 | **Диапазон цен** | ВЫСОКИЙ | Min/max цена для каждого товара |
| 7 | **Реакция на остатки** | ВЫСОКИЙ | ↑ при малых остатках, ↓ при избытке |
| 8 | **Реакция на спрос** | ВЫСОКИЙ | ↑ при высоком спросе, ↓ при низком |
| 9 | **По расписанию** | СРЕДНИЙ | Разные цены будни/выходные |
| 10 | **A/B-тестирование** | СРЕДНИЙ | С лимитом отклонения ±X% |

**Обязательно:** комбинирование стратегий + приоритеты при конфликте.

## 11. UI/UX требования

### Ключевой инсайт
> «Очень нравится интерфейс и структура индипы. Хотелось бы иметь ее копию по интерфейсу и функционалу с возможностью добавлять необходимые для себя источники данных, кастомные стратегии и кастомизации интерфейса и дашбордов.»

### Подход
- **Базовая структура — как Indeepa**
- **Кастомизация:** свои источники данных, свои стратегии, свои дашборды
- **Гибкий, с множеством настроек**

### Дашборд главной страницы
- Общая прибыль за период
- Количество изменений цен за день
- Товары с критично низкой маржой
- Активные акции
- Товары без стратегии

### Идеальный рабочий день (по описанию заказчика)
1. **Утро:** Общий отчёт по кабинету (заказы, маржа, выручка, отмены, выкупы)
2. **Далее:** Отчёт о наличии ошибок на товарах
3. **Далее:** Анализ акционных цен
4. **Система:** Автоматически меняет цены по установленным стратегиям

## 15. Сводка: что строим

### В одном предложении
**Кастомизируемый репрайсер для WB с фокусом на динамическое ценообразование по скорости продаж, защиту от out-of-stock, управление акционными порогами и глубокую аналитику — «Indeepa, но дешевле, точнее и гибче».**

### Отличия от конкурентов (наша ниша)
1. **Уникальная стратегия** — ценообразование по динамике 7д/14д (никто не делает так специфично)
2. **Точные данные** — прямая работа с API WB (проблема Indeepa — расхождения)
3. **Кастомизация** — свои стратегии, свои дашборды, свои источники данных
4. **Акционный бустинг** — умное поднятие медианы перед акциями
5. **Доступная цена** — дешевле Indeepa (от 15 900 ₽/мес)
